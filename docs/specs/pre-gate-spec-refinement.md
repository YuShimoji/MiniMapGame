# Pre-Gate Spec Refinement (Blocker Outside Work)

## 目的

Gate-1（P4道路手動検証）待ちの間に、実装を進めずに仕様の不確実性を下げる。
一問一答で決める前提で、論点を分離して整理する。

## 調査サマリ

- Gate-1は次実装のブロッカーだが、仕様整理は継続可能
- ドキュメント間で軽微な整合ズレがあり、優先して修正する価値がある
- 実装前に決めるべき論点（B/D/E/W-2）を質問単位に分解できる

## 一問一答の決定ログ

- 2026-03-09 Q1（E）: 道路幅テーブルの表記は `max,min` に統一する

## 一問一答キュー（実装前に確定）

1. B: 交差点形状の第一候補は「見た目優先」か「実装コスト優先」のどちらを取りますか？
2. D: 劣化トリガーは「探索距離」「発見数」「経過時間」のどれを主指標にしますか？
3. W-2: 入り江/岬の強さはプリセット固定値で開始しますか、WaterProfileで可変にしますか？

## メモ

- `BuildingPlacer` 側の道路/建物幅レンジは JSX由来の降順配列（例: 12→8）で実装済み。
- 表記上の誤解を避けるため、Eでは「値の正しさ」と「表の読みやすさ」を分けて扱う。
